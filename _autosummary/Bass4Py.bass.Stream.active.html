Stream.active¶

Stream.active¶

a channel is active (playing) or stalled. Can also check if a recording is in progress.

One of the following:

  • Bass4Py.constants.ACTIVE.STOPPED: The channel is not active.

  • Bass4Py.constants.ACTIVE.PLAYING: The channel is playing (or recording).

  • Bass4Py.constants.ACTIVE.PAUSED: The channel is paused.

  • Bass4Py.constants.ACTIVE.PAUSED_DEVICE: The channel’s device is paused.

  • Bass4Py.constants.ACTIVE.STALLED: Playback of the stream has been stalled due to a lack of sample data. Playback will automatically resume once there is sufficient data to do so.

When using this attribute with a decoding channel, Bass4Py.constants.ACTIVE.PLAYING will be returned while there is still data to decode. Once the end has been reached, Bass4Py.constants.ACTIVE.STOPPED will be returned. Bass4Py.constants.ACTIVE.STALLED is never returned for decoding channels; you can tell a decoding channel is stalled if get_data() returns less data than requested and this attribute still returns Bass4Py.constants.ACTIVE.PLAYING. The Bass4Py.constants.ACTIVE.PAUSED_DEVICE state can be the result of a Bass4Py.bass.OutputDevice.pause() call or of the device stopping unexpectedly (eg. a USB soundcard being disconnected). In either case, playback will be resumed by Bass4Py.bass.OutputDevice.start(). Syncs can be set via Bass4Py.bass.Channel.set_sync() to be notified when a channel reaches the end (Bass4Py.bass.syncs.End sync) or stalls/resumes (Bass4Py.bass.syncs.Stall sync) or pauses due to device failure (Bass4Py.bass.syncs.DeviceFail sync).
