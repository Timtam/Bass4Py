OutputDevice.init¶

OutputDevice.init(self, DWORD freq=44100, DWORD flags=0)¶

Initializes an output device.

Parameters:

  • freq (int) – Output sample rate.

  • flags (Bass4Py.constants.DEVICE) – a combination of the Bass4Py.constants.DEVICE flags

Returns:

success

Return type:

bool

Raises:

  • Bass4Py.exceptions.BassDxError – DirectX (or ALSA on Linux) is not installed.

  • Bass4Py.exceptions.BassDeviceError – device is invalid.

  • Bass4Py.exceptions.BassAlreadyError – The device has already been initialized. free() must be called before it can be initialized again.

  • Bass4Py.exceptions.BassDriverError – There is no available device driver.

  • Bass4Py.exceptions.BassBusyError – Something else has exclusive use of the device.

  • Bass4Py.exceptions.BassFormatError – The specified format is not supported by the device. Try changing the freq and flags parameters.

  • Bass4Py.exceptions.BassMemoryError – There is insufficient memory.

  • Bass4Py.exceptions.BassNo3DError – Could not initialize 3D support.

  • Bass4Py.exceptions.BassUnknownError – Some other mystery problem!

This function must be successfully called before using any sample, stream or MOD music functions. The recording functions may be used without having called this function. The Bass4Py.bass.BASS.device_buffer and Bass4Py.bass.BASS.device_update_period config options determine how much data is buffered for the device and how often it is updated.

Platform-specific

The sample format specified in the freq and flags parameters has no effect on the device output on iOS or OSX, and not on Windows unless VxD drivers are used (on Windows 98/95); with WDM drivers (on Windows XP/2000/Me/98SE), the output format is automatically set depending on the format of what is played and what the device supports, while on Vista and newer, the output format is determined by the user’s choice in the Sound control panel. On Linux, the output device will use the specified format if possible, but will otherwise use a format as close to it as possible. If the Bass4Py.constants.DEVICE.FREQ flag is specified on iOS or OSX, then the device’s output rate will be set to the freq parameter if possible. The Bass4Py.constants.DEVICE.FREQ flag has no effect on other platforms. flags attribute can be used to check what the output format actually is. When DirectSound output is used on Windows, BASS will not be generating the final output mix and so some BASS features will be unavailable, including sample rate conversion quality configuration, playback buffer bypassing, and access to the final mix. The Bass4Py.constants.DEVICE.CPSPEAKERS and Bass4Py.constants.DEVICE.SPEAKERS flags only have effect on pre-Vista Windows; the number of available speakers is always accurately detected otherwise. The Bass4Py.constants.DEVICE.STEREO flag is ignored on Windows and OSX. The Bass4Py.constants.DEVICE.DMIX flag is only available on Linux, and allows multiple applications to share the device (if they all use “dmix”). It may also be possible for multiple applications to use exclusive access if the device is capable of hardware mixing. If exclusive access initialization fails, the Bass4Py.constants.DEVICE.DMIX flag will automatically be tried; if that happens, it can be detected via Bass4Py.bass.OutputDevice.init_flags.
