Slide.set¶

Slide.set(self, Channel chan)¶

Sets up this synchronizer to a channel.

Parameters:

chan (Bass4Py.bass.Channel) – Any channel object or one of its subclasses.

Raises:

Bass4Py.exceptions.BassSyncError – Either a Bass4Py.bass.Sync.callback wasn’t provided or the required parameters weren’t set beforehand. Please read the documentation of the according sync object and read the raised exception for more details.

Multiple synchronizers may be used per channel, and they can be set before and while playing. Equally, synchronizers can also be removed at any time, using remove(). If the one_time attribute is used then the sync is automatically removed after its first occurrence. The method set_mix_time() (with threading disabled) can be used with Bass4Py.bass.syncs.End or Bass4Py.bass.syncs.Position / Bass4Py.bass.syncs.MusicPosition syncs to implement custom looping, by using Bass4Py.bass.Channel.set_position() in the callback. A mixtime sync can also be used to make DSP/FX changes at specific points, or change a Bass4Py.bass.Music channel’s attributes. The set_mix_time() method can also be useful with a Bass4Py.bass.syncs.SetPosition sync, to reset DSP states after seeking. Several of the sync types are triggered in the process of rendering the channel’s sample data; for example, Bass4Py.bass.syncs.Position and Bass4Py.bass.syncs.End syncs, when the rendering reaches the sync position or the end, respectively. Those sync types should be set before starting playback or pre-buffering (ie. before any rendering), to avoid missing any early sync events. A channel does not need to be playing for its Bass4Py.bass.syncs.DeviceFail and Bass4Py.bass.syncs.DeviceFormat syncs to be triggered but the device does need to be active, which means it needs to be playing other channels or the Bass4Py.bass.BASS.device_nonstop option needs to be enabled. With recording channels, Bass4Py.bass.syncs.Position syncs are triggered just before the callback receives the block of data containing the sync position.
