TODO¶

This project is still a work in progress. Thus, many features have not been implemented yet or need to be reworked at some point. You’ll find the list of TODOs loosely listed below. Feel free to open an issue on GitHub if you feel inspired or want to leave a comment upon one of the things mentioned here.

Todo

I’d really like to either have all those properties and methods available at module level, or at least direct access to an BASS object instead of always have to instanciate the class myself, similar to how twisted handles its reactor access. That’d be much more comfortable, since nothing within this class requires saving any object-specific niformation and the properties are global anyway, which might confuse one or another because of the way classes and objects usually contain local data only.

(The original entry is located in docstring of Bass4Py.bass.bass.BASS, line 12.)

Todo

The reset parameter to Bass4Py.bass.Channel.set_position() isn’t yet supported and requires a new method (reset?) for the channel class.

(The original entry is located in docstring of Bass4Py.bass.channel.Channel, line 1.)

Todo

3D properties need to be reworked. Right now, each call to their setters will result in BASS_Apply3D() being executed, which makes them non-atomic and might cause issues when changing multiple settings in quick succession. I also don’t like the naming scheme right now, it’s not intuitive and doesn’t refer to 3D settings by just looking at them.

(The original entry is located in docstring of Bass4Py.bass.channel.Channel, line 6.)

Todo

get_tags() requires custom implementations to properly return the plain tag data for all different tag formats. But we also do have the Tags extension which allows us to interpret at least ID3 tags properly. What should we do now then?

  • properly implement get_tags() plus Tags extension?

  • get rid of get_tags() completely and implement the Tags extension properly instead?

  • should we bind the Tags extension to the Channel object directly or handle it as an external class which can be executed by handing it the Channel as a parameter?

(The original entry is located in docstring of Bass4Py.bass.channel.Channel, line 14.)
